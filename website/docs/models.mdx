# Supported Models

What you can run depends on the [driver](./drivers) you choose.

- **`native` driver** — pie's own engine. Each architecture needs an
  explicit implementation. The list below is the registry in
  `pie/src/pie_driver/model/__init__.py`.
- **`vllm` / `sglang` drivers** — pie delegates inference. You get the
  full vLLM / SGLang model zoos. Caveat: some inferlet features (custom
  attention masks, page-trim, raw-logits sampling) are native-only.

## Architectures supported by the native driver

| Family | Pie module | HF `model_type` | Examples |
|---|---|---|---|
| **Llama 3.x** | `llama3.py` | `llama` (also `l4ma`) | Llama 3, 3.1, 3.2 (instruct + base); Mistral 7B (Llama-compatible). |
| **Qwen 2.x** | `qwen2.py` | `qwen2` | Qwen2, Qwen2.5 (all sizes, instruct + base). |
| **Qwen 3.x** | `qwen3.py` | `qwen3` | Qwen3 (incl. the 0.6B default). |
| **Gemma 2** | `gemma2.py` | `gemma2` | Gemma 2 (Google open weights). |
| **Gemma 3** | `gemma3.py` | `gemma3_text` | Gemma 3 text-only checkpoints. |
| **Gemma 4** | `gemma4.py` | `gemma4_text`, `gemma4` | Gemma 4 (text). |
| **Mistral 3** | `mistral3.py` | `mistral3` | Ministral-class checkpoints. |
| **OLMo 3** | `olmo3.py` | `olmo3` | AI2 OLMo 3. |
| **GPT-OSS** | `gpt_oss.py` | `gptoss`, `gpt_oss` | Community GPT-OSS variants. |
| **Dummy** | `dummy.py` | — | Used by the dummy driver only. |

`pie model list` reports `✓` for HuggingFace repos whose
`config.json` `model_type` resolves to one of these.

## Downloading

```bash
pie model download Qwen/Qwen3-0.6B                  # 0.6B default
pie model download Qwen/Qwen2.5-7B-Instruct
pie model download meta-llama/Llama-3.2-3B-Instruct
pie model download meta-llama/Llama-3.1-70B-Instruct
pie model download google/gemma-3-2b-it
```

```bash
pie model list           # show what's cached + which are pie-compatible
pie model remove <repo>  # delete from cache
```

## Configuring the model

```toml
[model.default]
hf_repo = "Qwen/Qwen3-0.6B"

[model.default.driver]
type = "native"
device = ["cuda:0"]

[model.default.driver.native]
gpu_mem_utilization = 0.8
```

Multi-GPU (tensor parallel × data parallel):

```toml
[model.default.driver]
type = "native"
device = ["cuda:0", "cuda:1", "cuda:2", "cuda:3"]
tensor_parallel_size = 4    # 4-way TP, 1× DP
```

If `device` has more entries than `tensor_parallel_size`, the leftovers
form data-parallel replicas (e.g. `device = [...8 GPUs]` with
`tensor_parallel_size = 4` runs as 2 DP groups of 4-way TP).

You can have multiple `[model.<name>]` blocks. The first one (or whatever
`server.primary_model` names) is the default; inferlets address the
others via `Model::load(name)`.

## Quantization

The `native` driver supports float and quantized weight dtypes; the
choice maps to a torchao config under the hood
(`pie_driver/quantization.py`).

```toml
[model.default.driver]
activation_dtype = "bfloat16"

[model.default.driver.native]
weight_dtype = "float8"     # auto / float32 / float16 / bfloat16 / int4 / int8 / float8
```

| Value | Notes |
|---|---|
| `"auto"` | Inherit `activation_dtype`. |
| `"float16"` / `"bfloat16"` / `"float32"` | Plain floating-point weights. |
| `"int8"` | int8 weight-only quantization (torchao). |
| `"int4"` | int4 weight-only quantization (torchao). |
| `"float8"` | float8 weights (torchao). |

`activation_dtype` (one level up under `[model.X.driver]`) is independent
— typically `bfloat16`.

vLLM and SGLang have their own quantization knobs; configure those in
the driver-specific subtable (`[model.X.driver.vllm]` /
`[model.X.driver.sglang]`).

## Adding model support

To support a new architecture under the native driver:

1. Add `pie/src/pie_driver/model/<name>.py` implementing the model.
2. Register it in `pie/src/pie_driver/model/__init__.py` with the matching
   HF `model_type`.

Or, for a one-off, point the model at the `vllm`/`sglang` driver — those
will accept anything in the upstream backend's zoo.

If the architecture isn't in the upstream backends either, please
[open an issue](https://github.com/pie-project/pie/issues) with the
HuggingFace link.
