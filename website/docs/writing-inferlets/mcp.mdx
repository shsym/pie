# MCP Integration

The [Model Context Protocol](https://modelcontextprotocol.io/) is an open
standard for exposing tools, resources, and prompt templates to LLM
applications. Pie ships with an MCP client so inferlets can discover and
call MCP servers without writing transport code.

## Connecting

```rust
use inferlet::mcp::client;

// Names of all servers the host has registered for this session.
let servers = client::available_servers();

// Open a session — the resource closes when dropped.
let session = client::connect(&servers[0])?;
```

The host (your `pie serve`, or a client that registered an MCP server via
`Client::register_mcp_server`) is responsible for spawning the actual MCP
servers; the inferlet just sees their names.

## Tools

```rust
// list_tools / list_resources / list_prompts return a JSON string
// (the raw MCP `tools/list` response). Parse it with serde_json.
let tools_json: String = session.list_tools()?;
let tools: serde_json::Value = serde_json::from_str(&tools_json)?;

for tool in tools["tools"].as_array().unwrap() {
    eprintln!("{}: {}", tool["name"], tool["description"]);
}

// Call a tool by name with JSON arguments. Returns a list of `content`.
use inferlet::mcp::types::Content;

let args = serde_json::json!({"query": "Rust release"}).to_string();
let result: Vec<Content> = session.call_tool("search", &args)?;

for chunk in result {
    match chunk {
        Content::Text(s)               => print!("{s}"),
        Content::Image(img)            => save_image(&img.data, &img.mime_type),
        Content::EmbeddedResource(r)   => handle_resource(&r),
    }
}
```

## Resources

```rust
let resources_json = session.list_resources()?;
let parsed: serde_json::Value = serde_json::from_str(&resources_json)?;

let uri = parsed["resources"][0]["uri"].as_str().unwrap();
let content: Vec<Content> = session.read_resource(uri)?;
```

## Prompts

```rust
let prompts_json = session.list_prompts()?;
let parsed: serde_json::Value = serde_json::from_str(&prompts_json)?;

let name = parsed["prompts"][0]["name"].as_str().unwrap();
let args = serde_json::json!({"topic": "Rust"}).to_string();
let rendered_json = session.get_prompt(name, &args)?;
```

`get_prompt` returns the raw MCP `prompts/get` JSON — the messages array is
under `result.messages`.

## Bridging MCP tools into the chat decoder

Wire MCP tools into Pie's [tool-calling](./tool-calling) flow by parsing
the MCP tool list, equipping the model with the schemas, and dispatching
`Event::ToolCall` to the session:

```rust
// 1) Build the equip_tools input from the MCP tool list.
let tools_json: serde_json::Value =
    serde_json::from_str(&session.list_tools()?)?;

let equip: Vec<String> = tools_json["tools"]
    .as_array().unwrap()
    .iter()
    .map(|t| t.to_string())
    .collect();
ctx.equip_tools(&equip)?;

ctx.user("Search for the latest Rust release").cue();

// 2) Stream events, dispatch ToolCall to MCP.
let mut events = ctx.generate(sampler).decode().with_tool_use();

while let Some(ev) = events.next().await? {
    if let inferlet::Event::ToolCall(name, args) = ev {
        let result = session.call_tool(&name, &args)?;
        let answer = render_content(&result);
        events.stream_mut().ctx().answer_tool(&name, &answer).cue();
    }
}
```

`render_content` is up to you — concatenate `Content::Text`, encode
images, etc.
