# Model & Context

Every inferlet starts with two objects:

- **`Model`** — a loaded LLM (weights + tokenizer), shared across inferlets.
- **`Context`** — a token sequence with an attached KV cache.

## Loading a model

```rust
use inferlet::{model::Model, runtime};

// Names of every model the engine has loaded
let names = runtime::models();

// Load by name (matches the [model.<name>] key in ~/.pie/config.toml)
let model = Model::load(names.first().ok_or("No models available")?)?;
```

The runtime owns model lifetimes — multiple inferlets can `Model::load`
the same name simultaneously without re-loading weights.

## Tokenizer

```rust
let tok = model.tokenizer();

// text → tokens
let ids: Vec<u32> = tok.encode("What is the capital of France?");

// tokens → text
let text: String = tok.decode(&ids)?;

// Vocabulary: (token_ids, raw_byte_sequences)
let (ids, bytes) = tok.vocabs();

// Special tokens (BOS, EOS, etc.)
let (sp_ids, sp_bytes) = tok.special_tokens();

// The split regex used during BPE pre-tokenization
let regex = tok.split_regex();
```

## Creating a context

```rust
use inferlet::Context;

let mut ctx = Context::new(&model)?;
```

A fresh `Context` is **anonymous** — its KV pages are released when the
inferlet exits.

### Persistent contexts

Save a context under a name to keep its KV pages alive across runs:

```rust
ctx.save("my-session")?;       // copy this context to a named slot
let snap = ctx.snapshot()?;    // serialize → string for offline storage
```

Later, reattach:

```rust
let ctx = Context::open(&model, "my-session")?;   // shared (still owned by registry)
let ctx = Context::take(&model, "my-session")?;   // move ownership to this inferlet
Context::delete(&model, "my-session")?;            // drop the named slot
```

`open` clones (copy-on-write). `take` removes the slot from the registry
and hands ownership to the caller.

## Filling a context

The instruct API (`system / user / assistant / cue`) is the high-level
way to add tokens — see [Chat & Instruct](./chat-instruct).

For raw input:

```rust
ctx.append(&token_ids);     // queue tokens in the SDK-side buffer
ctx.flush().await?;         // run a forward pass to commit pages
```

`flush()` drains the buffer, runs prefill, and commits new KV pages. It
does **not** sample — use [`generate()`](./generation) for that.

```
  ctx.append(t)        ctx.flush().await?         ctx.generate(s)
  ┌──────────┐   ┌──────────────────────┐   ┌──────────────────┐
  │ Buffer   │──►│ Prefill: KV computed,│──►│ Autoregressive   │
  │ tokens   │   │ pages committed      │   │ generation       │
  └──────────┘   └──────────────────────┘   └──────────────────┘
```

## Forking

`fork()` is a copy-on-write clone. Both contexts share committed KV pages
until the fork writes new tokens.

```rust
let mut child = ctx.fork()?;
child.user("…");
child.cue();
```

Fork is O(1) — no page copies. Useful for branching exploration (tree of
thought, best-of-N, etc.).

## Inspecting state

```rust
ctx.model();              // &Model
ctx.page_size();          // u32 — tokens per KV page
ctx.seq_len();            // u32 — total committed + working tokens
ctx.buffer();             // &[u32] — SDK-side buffered tokens (not yet flushed)
ctx.inner();              // &RawContext — escape hatch to the WIT resource
```

## Suspending and bidding

Every running context places a *bid* in the engine's market for KV pages.
The SDK manages this automatically (`generate()` sets a budget-exhausting
bid for you), but you can override:

```rust
ctx.bid(0.0);                                 // step out of contention
let _g = ctx.yield_bid(Duration::from_secs(2)); // RAII low-bid window
ctx.suspend()?;                               // return all pages to the pool
```

See `compute_bid` in `inferlet::context` for the formula behind the
default bid.
