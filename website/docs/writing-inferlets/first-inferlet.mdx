# Your First Inferlet

Build a minimal inferlet from scratch — project setup, WASM build, and a
real model invocation.

## Prerequisites

Add the WASM target:

```bash
rustup target add wasm32-wasip2
```

You also need a running Pie engine (or just use `pie run` which boots one
on the fly).

## Project setup

Create a Rust library crate:

```bash
cargo new --lib my-inferlet
cd my-inferlet
```

`Cargo.toml`:

```toml
[package]
name = "my-inferlet"
version = "0.1.0"
edition = "2024"

[lib]
crate-type = ["cdylib"]

[dependencies]
inferlet = "0.2"
serde = { version = "1.0", features = ["derive"] }

[profile.release]
lto = true
```

`Pie.toml` (sits next to `Cargo.toml`; the `inferlet::main` macro reads it
at compile time):

```toml
[package]
name = "my-inferlet"
version = "0.1.0"
description = "My first inferlet"
authors = ["Me <me@example.com>"]

[runtime]
core = "^0.2.0"

[parameters]
prompt = { type = "string", description = "User prompt" }
max_tokens = { type = "int", optional = true }
```

## Write the inferlet

`src/lib.rs`:

```rust
use inferlet::{Context, Event, Result, inference::Sampler, model::Model, runtime};
use serde::{Deserialize, Serialize};

#[derive(Deserialize)]
struct Input {
    prompt: String,
    #[serde(default = "default_max_tokens")]
    max_tokens: usize,
}

fn default_max_tokens() -> usize { 256 }

#[derive(Serialize)]
struct Output {
    text: String,
}

#[inferlet::main]
async fn main(input: Input) -> Result<Output> {
    // Pick the first model the engine has loaded.
    let models = runtime::models();
    let name = models.first().ok_or("No models available")?;
    let model = Model::load(name)?;

    // Build a chat context.
    let mut ctx = Context::new(&model)?;
    ctx.system("You are a helpful assistant.");
    ctx.user(&input.prompt);
    ctx.cue();

    // Stream the response. `decode()` turns raw tokens into chat events.
    let mut events = ctx
        .generate(Sampler::top_p(0.6, 0.95))
        .with_max_tokens(input.max_tokens)
        .decode();

    let mut text = String::new();
    while let Some(event) = events.next().await? {
        match event {
            Event::Text(s) => { print!("{s}"); text.push_str(&s); }
            Event::Done(full) => { text = full; break; }
            _ => {}
        }
    }

    Ok(Output { text })
}
```

### What's happening

1. **`#[inferlet::main]`** generates the WASM entry point and a JSON
   bridge. Because `Input`/`Output` aren't `String`, the macro wires up
   `serde_json::from_str` / `to_string` for you.
2. **`Model::load(name)`** binds to a model the engine has loaded — the
   name is the `[model.<name>]` key from `~/.pie/config.toml`.
3. **`Context::new(&model)`** allocates a fresh KV cache context.
4. **`ctx.system / user / cue`** templates a chat conversation.
5. **`ctx.generate(sampler).with_max_tokens(n).decode()`** streams tokens
   through the model's decoder, surfacing text chunks via
   [`Event`](./generation#eventstream).

## Build

```bash
cargo build --target wasm32-wasip2 --release
```

You'll get `target/wasm32-wasip2/release/my_inferlet.wasm`.

:::tip
`bakery build <input>` (from the [Bakery toolchain](./building-publishing))
wraps the cargo step and packages the manifest in one shot.
:::

## Run

`pie run --path` needs both the WASM and the manifest:

```bash
pie run \
  --path target/wasm32-wasip2/release/my_inferlet.wasm \
  --manifest ./Pie.toml \
  --prompt "What is the capital of France?" \
  --max-tokens 64
```

The CLI flags after the manifest become the JSON input
(`{"prompt": "…", "max_tokens": 64}`), which `serde_json::from_str` lifts
into your `Input` struct.

## Notes

### Error handling

`Result<T>` aliases `Result<T, String>`. Any `Err(s)` you return becomes
the `Error` event the client receives.

```rust
let model = Model::load(name)?;   // propagates Err("...") on failure
```

### Debugging

`eprintln!()` writes to stderr, which `pie run` and clients receive as
`Stderr` events.

```rust
eprintln!("[debug] loaded model {name}, generating…");
```
