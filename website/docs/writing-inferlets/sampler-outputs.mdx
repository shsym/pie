# Sampler Outputs

The `Sampler` enum supports two kinds of samplers:

- **Token samplers** — they pick a token (`greedy`, `top_p`, `top_k`,
  `multinomial`, …). Covered in [Generation](./generation).
- **Output samplers** — they hand the inferlet *data* (a distribution,
  raw logits, logprobs, entropy) instead of choosing a token.

Output samplers don't write into the KV cache — they just expose the
model's view at the requested positions. Use them for best-of-N,
reranking, scoring, beam search, custom samplers, uncertainty-aware
control flow, etc.

## The `Output` shape

Every forward pass returns an `Output`:

```rust
pub struct Output {
    pub slots: Vec<SlotOutput>,
    pub spec_tokens: Vec<u32>,        // next-iter draft tokens (spec channel)
    pub spec_positions: Vec<u32>,
}

pub enum SlotOutput {
    Token(u32),                                  // multinomial / top-k / top-p / min-p / …
    Distribution((Vec<u32>, Vec<f32>)),          // dist
    Logits(Vec<u8>),                             // raw_logits — packed f32 bytes
    Logprobs(Vec<f32>),                          // logprob (len 1) / logprobs (len K)
    Entropy(f32),                                // entropy
    Embedding(Vec<u8>),                          // (placeholder)
}
```

Each `slots[i]` corresponds to the i-th `forward_pass.sampler(...)` call
in attachment order. A single forward pass can mix sampler kinds — e.g.,
`top_p` for the next token *and* `entropy` on the same position for an
uncertainty signal.

When you use `ctx.generate(...)` and the sampler is a single token
sampler, the SDK consumes the slot list for you. For output samplers
you'll want to drive the forward pass yourself — see
[Custom forward pass](./custom-forward-pass).

## Convenience accessors

For the common single-sampler case, there are one-liners on `Output`:

```rust
out.first_token();         // Option<u32>
out.tokens();              // Iterator<Item = u32> over every Token slot
out.first_distribution();  // Option<(&[u32], &[f32])>
out.first_logits();        // Option<&[u8]>     — packed f32 bytes
out.first_logprobs();      // Option<&[f32]>    — length 1 or K
out.first_entropy();       // Option<f32>
```

Returning `None` means the first slot wasn't of that variant — handle it
or `match` on `out.slots[0]` directly.

## Sampler constructors for outputs

```rust
use inferlet::inference::Sampler;

Sampler::distribution(temperature, k)    // top-k token IDs + probs (post-softmax, with temp)
Sampler::raw_logits()                    // pre-softmax, untemperatured logits as packed f32 bytes
Sampler::logprob(token_id)               // log p(token_id | context); length-1 Vec<f32>
Sampler::logprobs(vec![id1, id2, ...])   // log p for each candidate; length-K Vec<f32>
Sampler::entropy()                       // Shannon entropy of unscaled distribution
```

## Examples

### Top-k distribution + same-step entropy

```rust
use inferlet::inference::{ForwardPass, Sampler, SlotOutput};
use inferlet::ForwardPassExt;

let pass = ForwardPass::new(&model);
pass.context(&ctx);
pass.input_tokens(&buffered, &positions);

// Two slots at the same output position.
let last = (positions.len() - 1) as u32;
pass.sampler(&[last], Sampler::distribution(1.0, 32));
pass.sampler(&[last], Sampler::entropy());

let out = pass.execute_async().await?;

if let SlotOutput::Distribution((ids, probs)) = &out.slots[0] {
    // pick from the top 32 with custom logic
}
if let SlotOutput::Entropy(h) = &out.slots[1] {
    if *h > 3.0 { /* high uncertainty — branch off */ }
}
```

### Yes/no scoring

```rust
let yes = ctx.model().tokenizer().encode(" yes");
let no  = ctx.model().tokenizer().encode(" no");

pass.sampler(
    &[last],
    Sampler::logprobs(vec![yes[0], no[0]]),
);

let out = pass.execute_async().await?;
let lp = out.first_logprobs().unwrap();   // [logp_yes, logp_no]
let prefer_yes = lp[0] > lp[1];
```

### Raw logits

```rust
pass.sampler(&[last], Sampler::raw_logits());

let out = pass.execute_async().await?;
let bytes = out.first_logits().unwrap();
let logits: &[f32] = bytemuck::cast_slice(bytes);     // length = vocab_size
let argmax = logits.iter().enumerate()
    .max_by(|a, b| a.1.partial_cmp(b.1).unwrap()).unwrap().0;
```

`raw_logits` is *pre-softmax* and has no temperature applied, so it
mirrors the model's native distribution exactly.

## Why slot lists?

`Output` is a `record` not an enum: a single forward pass can mix
arbitrary sampler kinds without losing data. This makes patterns like
"sample a token *and* compute the entropy at that position" cheap — both
slots share one forward pass.

## Caveats

- The KV cache is **not** updated by output samplers. If you also want
  the model to commit a token at this position, attach a token sampler
  to the same slot.
- `embedding` is reserved but not yet wired end-to-end.
- The `f32` byte buffers (`raw_logits`, `embedding`) are *native-endian*.
  Decode with `bytemuck` or equivalent.
