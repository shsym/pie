# I/O & Messaging

Inferlets run inside a WASI sandbox. WASI gives them HTTP, filesystem,
timers, and `wstd::task` — Pie layers a `messaging` interface on top for
inter-inferlet and host coordination.

## HTTP client

```rust
use inferlet::wstd::http::{Client, Method, Request};
use inferlet::wstd::io::{AsyncRead, empty};

async fn fetch(url: &str) -> Result<Vec<u8>> {
    let client = Client::new();
    let req = Request::builder()
        .uri(url)
        .method(Method::GET)
        .body(empty())?;
    let resp = client.send(req).await?;

    let mut body = resp.into_body();
    let mut buf = Vec::new();
    body.read_to_end(&mut buf).await?;
    Ok(buf)
}
```

Use it for fetching documents, calling external APIs, or downloading
resources at request time.

## HTTP server inferlets

A separate entry-point macro turns an inferlet into an HTTP server. Used
e.g. by `inferlets/http-server/`:

```rust
use wstd::http::body::IncomingBody;
use wstd::http::server::{Finished, Responder};
use wstd::http::{IntoBody, Request, Response, StatusCode};

#[wstd::http_server]
async fn main(req: Request<IncomingBody>, res: Responder) -> Finished {
    match req.uri().path() {
        "/" => res.respond(Response::new("hello".into_body())).await,
        _   => res.respond(
                  Response::builder().status(StatusCode::NOT_FOUND)
                      .body("404".into_body()).unwrap(),
              ).await,
    }
}
```

Run it with the `pie http` command (currently exposed by the runtime, not
the top-level `pie` CLI — see the inferlet's README).

### SSE / streaming responses

```rust
use wstd::http::body::BodyForthcoming;
use wstd::io::AsyncWrite;

let stream = Response::builder()
    .header("Content-Type", "text/event-stream")
    .body(BodyForthcoming).unwrap();
let mut body = res.start_response(stream);

for i in 0..5 {
    let frame = format!("data: {{\"i\":{i}}}\n\n");
    body.write_all(frame.as_bytes()).await?;
    body.flush().await?;
    wstd::task::sleep(Duration::from_millis(100)).await;
}
Finished::finish(body, Ok(()), None)
```

## File I/O

Standard `std::fs` works inside the WASI sandbox:

```rust
let s = std::fs::read_to_string("/data/in.txt")?;
std::fs::write("/data/out.json", &json)?;
```

Filesystem access is gated by the runtime — only directories the host
has explicitly granted are visible.

## Time

```rust
use wstd::time::{Duration, Instant};

wstd::task::sleep(Duration::from_secs(1)).await;
let t0 = Instant::now();
// …work…
let dt = t0.elapsed();
```

## Broadcast / subscribe

Topic-based fan-out: every subscriber sees every broadcast.

```rust
use inferlet::{messaging, SubscriptionExt};

// Publisher
messaging::broadcast("events", "new-data");

// Subscriber
let sub = messaging::subscribe("events");
loop {
    if let Some(msg) = sub.get_async().await {
        eprintln!("got {msg}");
    }
}
sub.unsubscribe();
```

`SubscriptionExt::get_async()` is the async polled receive. The raw
`subscription.get()` is non-blocking and returns `Option<String>`.

Used by `inferlets/agent-swarm/` to coordinate worker generations.

## Queues (push / pull)

Work-queue style: each `pull` consumes one message; topic order is
preserved.

```rust
use inferlet::{messaging, FutureStringExt};

// Producer
messaging::push("jobs", &job_payload);

// Consumer
let fut = messaging::pull("jobs");
let msg: Option<String> = fut.wait_async().await;
```

`FutureStringExt::wait_async()` resolves the WIT `future-string` handle.

## Concurrency

`wstd::task::spawn` runs tasks concurrently within an inferlet (e.g.,
parallel HTTP fetches alongside generation):

```rust
let h1 = wstd::task::spawn(async { fetch("https://a").await });
let h2 = wstd::task::spawn(async { fetch("https://b").await });
let (a, b) = (h1.await?, h2.await?);
```

## At a glance

| Pattern | Purpose |
|---|---|
| `wstd::http::Client` | Fetch external HTTP resources. |
| `#[wstd::http_server]` | Expose an HTTP/SSE endpoint from an inferlet. |
| `std::fs` | Read/write within the sandbox. |
| `messaging::broadcast / subscribe` | Topic fan-out — every subscriber sees every msg. |
| `messaging::push / pull` | Work-queue, single-consumer-per-message. |
| `wstd::task::spawn` | Concurrent tasks inside one inferlet. |
