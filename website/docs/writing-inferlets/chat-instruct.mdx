# Chat & Instruct templates

`Context` exposes the chat template directly — the model-specific format
(role tags, EOS, thinking tokens, etc.) is applied automatically. You
write the conversation; the SDK emits the right tokens.

## Building a conversation

```rust
ctx.system("You are a helpful assistant.");
ctx.user("What is the capital of France?");
ctx.cue();              // hand off to the model
```

| Method | Description |
|---|---|
| `ctx.system(text)` | Add a system message. |
| `ctx.user(text)` | Add a user message. |
| `ctx.assistant(text)` | Add a pre-filled assistant turn. |
| `ctx.cue()` | Mark the current position as the model's start. |
| `ctx.seal()` | Close the current assistant turn. |

All five return `&mut Self` so you can chain.

### Multi-turn

```rust
ctx.system("You are a math tutor.");

ctx.user("What is 2+2?").cue();
let answer = ctx.generate(sampler).with_max_tokens(64).collect_text().await?;
ctx.seal();              // close the assistant turn

ctx.user("Why?").cue();
let explain = ctx.generate(sampler).with_max_tokens(256).collect_text().await?;
ctx.seal();
```

`collect_text()` consumes the stream and writes the produced text into
the context, so the next turn already sees it; `seal()` just marks the
turn boundary.

## The decoder

Raw tokens come out of the model in the chat template's wire format
(thinking tags, tool-call syntax, EOS). The `Decoder` translates that
into a normalized `Event` enum so downstream code stays
model-agnostic.

```
 Encoding (Context):                 Decoding (Decoder):

 ctx.system("…")  ──┐                tokens ──► Decoder ──► Events
 ctx.user("…")    ──┤── template               ├── Text("…")
 ctx.cue()        ──┘   formatting             ├── Thinking("…")
        │                                       ├── ToolCall(name, args)
        ▼                                       └── Done(full_text)
 model-specific tokens             model-agnostic events
```

Use it via `EventStream`:

```rust
use inferlet::{Event, inference::Sampler};

let mut events = ctx
    .generate(Sampler::top_p(0.6, 0.95))
    .with_max_tokens(256)
    .decode()
    .with_reasoning()       // detect <think>…</think>
    .with_tool_use();       // detect tool calls
```

### Event variants

```rust
pub enum Event {
    Token,                          // raw token, no semantic meaning
    Text(String),                   // assistant text chunk
    Thinking(String),               // chunk inside a reasoning block
    ThinkingDone(String),           // reasoning block closed (full text)
    ToolCall(String, String),       // (tool_name, arguments_json)
    Done(String),                   // generation complete (full response)
}
```

### Decoder modes

| Mode | Detects |
|---|---|
| Default | `Text`, `Done` |
| `.with_reasoning()` | + `Thinking`, `ThinkingDone` |
| `.with_tool_use()` | + `ToolCall` |

The modes are additive.

### Standalone decoder

If you're driving a `TokenStream` manually, build a `Decoder` from the
model with `ModelExt`:

```rust
use inferlet::ModelExt;

let mut decoder = model.decoder()
    .with_reasoning()
    .with_tool_use();

let mut stream = ctx.generate(sampler).with_max_tokens(512);
while let Some(tokens) = stream.next().await? {
    match decoder.feed(&tokens)? {
        Event::Text(s) => print!("{s}"),
        Event::Done(_) => break,
        _ => {}
    }
}
```

`model.decoder()` and `EventStream` produce the same `Event`s — pick the
one that fits your control flow.

## Example: chat with reasoning

```rust
use inferlet::{Context, Event, Result, inference::Sampler, model::Model, runtime};
use serde::{Deserialize, Serialize};

#[derive(Deserialize)]
struct Input { prompt: String }

#[derive(Serialize)]
struct Output { thinking: String, text: String }

#[inferlet::main]
async fn main(input: Input) -> Result<Output> {
    let model = Model::load(runtime::models().first().ok_or("No models")?)?;
    let mut ctx = Context::new(&model)?;

    ctx.system("You are a helpful assistant.");
    ctx.user(&input.prompt);
    ctx.cue();

    let mut events = ctx
        .generate(Sampler::top_p(0.6, 0.95))
        .with_max_tokens(512)
        .decode()
        .with_reasoning();

    let mut thinking = String::new();
    let mut text = String::new();
    while let Some(event) = events.next().await? {
        match event {
            Event::Thinking(s)     => { eprint!("{s}"); thinking.push_str(&s); }
            Event::ThinkingDone(_) => { eprintln!(); }
            Event::Text(s)         => { print!("{s}"); text.push_str(&s); }
            Event::Done(full)      => { text = full; break; }
            _ => {}
        }
    }

    Ok(Output { thinking, text })
}
```
