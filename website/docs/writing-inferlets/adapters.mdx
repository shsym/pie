# Adapters & Fine-tuning

Pie supports runtime LoRA adapter management and a zeroth-order
optimization (Zo) interface for gradient-free updates from inside an
inferlet.

## Adapters

A `Adapter` is a named, model-bound LoRA overlay. Adapters are scoped to a
loaded model.

```rust
use inferlet::adapter::Adapter;

// Create a new named adapter for this model.
let adapter = Adapter::create(&model, "my-adapter")?;

// Open an existing one (returns Option).
let adapter = Adapter::open(&model, "my-adapter").ok_or("not found")?;

// Fork into a new named copy.
let v2 = adapter.fork("my-adapter-v2");

// Persist to / from disk.
adapter.save("/data/adapters/my-adapter.bin")?;
adapter.load("/data/adapters/my-adapter.bin")?;

// Drop it from the registry.
adapter.destroy();
```

## Using an adapter at inference time

Two ways:

**Through the `TokenStream` builder:**

```rust
let text = ctx
    .generate(sampler)
    .with_adapter(&adapter)
    .with_max_tokens(256)
    .collect_text()
    .await?;
```

**Or attach it directly to a `ForwardPass`** (when driving the forward
pass yourself — see [Custom forward pass](./custom-forward-pass)):

```rust
use inferlet::inference::ForwardPass;

let pass = ForwardPass::new(&model);
pass.context(&ctx);
pass.adapter(&adapter);
pass.input_tokens(&tokens, &positions);
let out = pass.execute_async().await?;
```

## Zeroth-order optimization (Zo)

`zo` is a gradient-free trainer that perturbs an adapter, evaluates the
loss, and updates via a CMA-ES-like rule. Useful when you can't
backprop (e.g., from inside the WASM sandbox) but still want online
adaptation.

### Initialize

```rust
use inferlet::zo;

zo::initialize(
    &adapter,
    16,       // rank: LoRA rank
    1.0,      // alpha: LoRA scaling
    8,        // population_size: perturbations per step
    0.5,      // mu_fraction: top fraction kept
    0.01,     // initial_sigma: starting perturbation scale
)?;
```

### Per-step loop

For each step you evaluate the population at perturbed seeds, then update.

```rust
let mut scores: Vec<f32> = Vec::new();
let mut seeds:  Vec<i64> = Vec::new();

for member in 0..population_size {
    let seed = generate_seed(member);

    // Build a forward pass and tag it with the perturbation seed.
    let pass = ForwardPass::new(&model);
    pass.context(&ctx);
    pass.adapter(&adapter);
    zo::adapter_seed(&pass, seed);

    pass.input_tokens(&tokens, &positions);
    let out = pass.execute_async().await?;

    scores.push(score_from_output(&out));
    seeds.push(seed);
}

zo::update(&adapter, &scores, &seeds, /* max_sigma */ 0.1)?;
```

`zo::adapter_seed` tags a single forward pass with the perturbation seed
to evaluate; `zo::update` consumes the population and advances the
distribution.

The `inferlets/raw-logits-demo/` and related examples show a complete
optimization loop.
