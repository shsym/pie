# Bakery

`bakery` is the developer-side CLI for inferlets. It wraps the cargo /
npm build for you, talks to the Pie registry, and produces packaged
WASM artifacts.

```bash
pip install -e sdk/tools/bakery        # from a Pie checkout
```

When developing inside a Pie checkout, set `PIE_SDK` so bakery can find
the SDK libraries:

```bash
export PIE_SDK=/path/to/pie/sdk
```

## Top-level commands

```
bakery [COMMAND]
  create        Scaffold a new inferlet project
  build         Build a project (Rust crate or JS/TS) to WASM
  login         Authenticate with the Pie Registry (GitHub OAuth)
  inferlet      Manage published inferlets (search / info / publish)
```

## bakery create

```bash
bakery create my-inferlet              # Rust template (default)
bakery create my-inferlet --ts         # TypeScript template
bakery create my-inferlet -o ./out     # custom output directory
```

The template lays down `Cargo.toml` (or `package.json`), `Pie.toml`, and
a working `lib.rs` / `index.ts` against the current SDK.

## bakery build

```bash
bakery build <input> -o <output.wasm> [--debug]
```

| Argument | Description |
|---|---|
| `<input>` | Path to a project directory (Rust: contains `Cargo.toml`; JS/TS: contains `package.json`) or a single `.js`/`.ts` file. |
| `-o, --output <FILE>` | Where to write the `.wasm`. **Required**. |
| `--debug` | Debug build (JS/TS only — includes source maps). |

```bash
bakery build ./my-rust-inferlet  -o build/my-inferlet.wasm
bakery build ./my-ts-inferlet    -o build/my-inferlet.wasm
bakery build ./index.ts          -o build/single-file.wasm
```

The platform is auto-detected from the input. For Rust projects bakery
just runs the right `cargo build --target wasm32-wasip2 --release`
under the hood.

## bakery login

```bash
bakery login
```

Opens a GitHub OAuth flow. The token is stored in `~/.pie/auth/registry.json`
and used by `bakery inferlet publish`.

## bakery inferlet

```
bakery inferlet
  search <query>       Search published inferlets
  info <name>          Show metadata for a published inferlet
  publish <path>       Build and publish an inferlet directory
```

```bash
bakery inferlet search agent
bakery inferlet info text-completion
bakery inferlet publish ./my-inferlet
```

`bakery inferlet publish` reads `Pie.toml`, builds, then uploads the
`.wasm` + manifest to the registry.

## End-to-end flow

```bash
# 1) scaffold (and replace lib.rs with the docs example)
bakery create my-inferlet
cd my-inferlet

# 2) iterate locally
bakery build . -o build/my-inferlet.wasm
pie run --path build/my-inferlet.wasm --manifest Pie.toml --prompt "hi"

# 3) publish
bakery login
bakery inferlet publish .

# 4) anyone can run it
pie run my-inferlet --prompt "hi"
```

## Requirements

| Platform | What you need |
|---|---|
| Rust inferlets | `cargo`, `rustup target add wasm32-wasip2` |
| JS / TS inferlets | Node.js v18+, `npm`/`npx` |
