# Server Mode

`pie run` is one-shot. For everything else — drives multiple clients, keeps
state across requests, gets monitored — start a long-lived server.

## Start the server

```bash
pie serve
```

Output:

```
╭─ Pie Engine ─────────────────────────────────╮
│ Host         127.0.0.1:8080                  │
│ Model        default (Qwen/Qwen3-0.6B)       │
│ Driver       native                          │
│ Device       cuda:0                          │
╰──────────────────────────────────────────────╯

✓ Server ready at ws://127.0.0.1:8080
```

The server is now accepting WebSocket clients.

## Monitor mode

A real-time TUI dashboard:

```bash
pie serve -m
```

It shows the running model, batch utilization, throughput, and per-process
status. Press `q` to exit the TUI; the server keeps running in the
background until you `Ctrl-C`.

## Common flags

| Option | Description |
|--------|-------------|
| `-c`, `--config <PATH>` | Use a non-default config file. |
| `--host <HOST>` | Override `[server].host`. |
| `--port <PORT>` | Override `[server].port`. |
| `--no-auth` | Disable authentication for this run. |
| `-v`, `--verbose` | Verbose logging. |
| `-m`, `--monitor` | Launch the TUI alongside the server. |
| `--no-snapshot` | Disable the host-side Python snapshot optimization. |

```bash
# Custom port, no auth (dev)
pie serve --port 9000 --no-auth -v

# Alternate config
pie serve -c ./dev.toml
```

See the [CLI reference](../cli#serve) for the full list.

## Connecting clients

```python
from pie_client import PieClient, ParsedPrivateKey, Event

async with PieClient("ws://127.0.0.1:8080") as client:
    # Skip the key arg if [auth].enabled = false
    key = ParsedPrivateKey.from_file("~/.ssh/id_ed25519")
    await client.authenticate("alice", key)

    proc = await client.launch_process(
        "text-completion@0.1.0",
        input={"prompt": "hello"},
    )
    while True:
        event, value = await proc.recv()
        if event == Event.Stdout: print(value, end="")
        elif event == Event.Return: break
```

The full client API is in [Client basics](./client-basics) and the
[Python](../client-api/python) / [JS](../client-api/javascript) /
[Rust](../client-api/rust) references.

## Authorizing clients

When `[auth].enabled = true`, only registered users can connect. Add a
public key:

```bash
cat ~/.ssh/id_ed25519.pub | pie auth add alice laptop
pie auth list
```

`pie auth remove alice [key_name]` revokes a single key or the whole user.

## Dummy backend (no GPU)

For protocol-level testing without loading real weights, set the driver to
`dummy`:

```bash
pie config set model.default.driver.type dummy
pie serve
```

The engine returns random tokens but every other code path runs.

## Graceful shutdown

`Ctrl-C`:

```
^C
✓ Shutdown complete
```

Pie stops accepting new connections, lets running processes finish, tears
down driver workers, and exits.
