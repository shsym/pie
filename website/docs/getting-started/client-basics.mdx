# Client Basics

How to connect to a running Pie server and drive inferlets from your code.

## Prerequisites

- A Pie server running (`pie serve`).
- A client library: `pie-client` (Python), `@pie/client` (JS), or
  `pie-client` (Rust crate).

## The lifecycle

Every client follows the same pattern:

1. Connect to `ws://host:port`.
2. Authenticate.
3. Launch a *process* (a running inferlet) by `name@version`, passing JSON input.
4. Receive a stream of events until `Return` (success) or `Error` (abort).

The terminal events for a process are `Return` and `Error`. After either,
the server closes the per-process event stream.

## Python

```bash
pip install pie-client
```

```python
import asyncio
from pie_client import PieClient, ParsedPrivateKey, Event

async def main():
    async with PieClient("ws://127.0.0.1:8080") as client:
        # If [auth].enabled = false on the server, you can pass key=None.
        key = ParsedPrivateKey.from_file("~/.ssh/id_ed25519")
        await client.authenticate("alice", key)

        proc = await client.launch_process(
            "text-completion@0.1.0",
            input={"prompt": "What is the capital of France?",
                   "max_tokens": 64},
        )

        while True:
            event, value = await proc.recv()
            if event == Event.Stdout:
                print(value, end="", flush=True)
            elif event == Event.Return:
                # value is the inferlet's JSON-encoded Output
                print("\n[done]", value)
                break
            elif event == Event.Error:
                print("\n[error]", value)
                break

asyncio.run(main())
```

### Signaling a running process

```python
proc = await client.launch_process("chat-loop@0.1.0")
await proc.signal("hello")          # → inferlet receives a string signal
await proc.transfer_file(b"...")    # → inferlet receives a binary blob
```

### Uploading a local build

```python
await client.install_program("./build.wasm", "./Pie.toml")
proc = await client.launch_process("my-inferlet@0.1.0", input={...})
```

## JavaScript

```bash
npm install @pie/client
```

```javascript
import { PieClient } from '@pie/client';

const client = new PieClient("ws://127.0.0.1:8080");
await client.connect();

// The JS client only exposes token-based auth today.
// For public-key auth, use the Python or Rust client.
await client.authByToken(process.env.PIE_TOKEN);

const proc = await client.launchProcess(
  "text-completion@0.1.0",
  { prompt: "Hello", max_tokens: 64 },
);

while (true) {
  const { event, value } = await proc.recv();
  if (event === "stdout") process.stdout.write(value);
  else if (event === "return") { console.log("\n[done]", value); break; }
  else if (event === "error")  { console.error("\n[error]", value); break; }
}

client.close();
```

## Rust

```toml
[dependencies]
pie-client = "*"
tokio = { version = "1", features = ["full"] }
anyhow = "1"
```

```rust
use pie_client::{Client, ParsedPrivateKey, ProcessEvent};

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    let client = Client::connect("ws://127.0.0.1:8080").await?;
    let key = ParsedPrivateKey::from_file("~/.ssh/id_ed25519")?;
    client.authenticate("alice", &Some(key)).await?;

    let mut proc = client
        .launch_process(
            "text-completion@0.1.0".into(),
            r#"{"prompt":"Hello","max_tokens":64}"#.into(),
            true,
            None,
        )
        .await?;

    loop {
        match proc.recv().await? {
            ProcessEvent::Stdout(s) => print!("{s}"),
            ProcessEvent::Return(v) => { println!("\n[done] {v}"); break; }
            ProcessEvent::Error(e)  => { eprintln!("\n[error] {e}"); break; }
            _ => {}
        }
    }

    client.close().await?;
    Ok(())
}
```

## Events

Every client surfaces the same six event kinds:

| Event | Payload | Meaning |
|-------|---------|---------|
| `Stdout` / `"stdout"` | text | Streamed stdout from the inferlet. |
| `Stderr` / `"stderr"` | text | Streamed stderr. |
| `Message` / `"message"` | text | A user-defined message from the inferlet. |
| `File` / `"file"` | bytes | A file the inferlet sent back (chunked, reassembled by the client). |
| `Return` / `"return"` | JSON text | Final `Output` — the process exited cleanly. |
| `Error` / `"error"` | text | The process aborted. |

Both `Return` and `Error` are terminal: don't call `recv()` again on the
same process after either one fires.

## Detached / reattach

`launch_process` returns a handle. Drop it, reconnect, call
`attach_process(process_id)`, and resume reading. While disconnected, the
server buffers events for you.

```python
pid = proc.process_id
# … later, possibly from a different host …
async with PieClient(uri) as c:
    await c.authenticate("alice", key)
    proc = await c.attach_process(pid)
    event, value = await proc.recv()
```

## Authentication notes

- `[auth].enabled = false`: Python/Rust can pass `private_key=None`/`&None`,
  JS skips auth entirely.
- `[auth].enabled = true`: register a key with
  `cat key.pub | pie auth add <user> <key_name>`, then sign in with the
  matching private key.
