# Installation

Pie supports Linux (CUDA) and macOS (Apple Silicon / Metal). Pick the path
that matches how you want to run it.

## Quick install (PyPI)

The fastest way to get started:

```bash
# Linux + NVIDIA GPU (CUDA 12.6 toolchain)
pip install "pie-server[cu126]"

# Linux + NVIDIA GPU (CUDA 12.8 toolchain)
pip install "pie-server[cu128]"

# macOS (Apple Silicon)
pip install "pie-server[metal]"
```

The extra you pick determines which Torch wheel index (and which kernel
backend) gets pulled in.

### Optional driver backends

Pie ships with a built-in driver (`type = "native"`) and can also delegate
inference to vLLM or SGLang as alternative drivers:

```bash
pip install "pie-server[cu128,vllm]"     # vLLM driver
pip install "pie-server[sglang]"         # SGLang driver
```

See [CLI configuration](../cli#configuration-file) for how to point a model
block at a specific driver.

## Install from source

For development or to track the tip:

```bash
git clone https://github.com/pie-project/pie.git && cd pie/pie

# uv resolves the right Torch / FlashInfer / kernel set per extra.
uv sync --extra cu128            # or cu126 / metal / vllm / sglang
```

When working from a source checkout, prefix commands with `uv run`:

```bash
uv run pie config init
uv run pie serve
```

## First-time setup

```bash
pie config init
```

`pie config init` writes a default `~/.pie/config.toml` and downloads the
Python 3.14 inferlet runtime into `~/.pie/`. The TOML it writes:

```toml
[server]
host = "127.0.0.1"
port = 8080

[auth]
enabled = false

[model.default]
hf_repo = "Qwen/Qwen3-0.6B"

[model.default.driver]
type = "native"
device = ["cuda:0"]   # or ["mps"] / ["cpu"] depending on the host
```

See [CLI configuration](../cli#configuration-file) for the full schema.

## Download a model

```bash
pie model download Qwen/Qwen3-0.6B
```

`pie model list` reports compatibility — Pie-supported architectures get a
`✓`. Other compatible repos:

```bash
pie model download Qwen/Qwen2.5-7B-Instruct
pie model download meta-llama/Llama-3.2-1B-Instruct
```

Point the config at a different model:

```bash
pie config set model.default.hf_repo Qwen/Qwen2.5-7B-Instruct
```

## Verify the install

```bash
pie run text-completion --prompt "What is the capital of France?"
```

`pie run` boots a one-shot engine, runs the inferlet, prints its output,
then exits. The first run is slower because of JIT/kernel warmup.

## Doctor

If something looks wrong, ask `pie doctor`:

```bash
pie doctor
```

It reports your platform, GPU/MPS availability, key Python package
versions, and whether the config + default model are present.
