# Drivers

The Pie *driver* is the inference backend a model runs on. Pie ships four
drivers and lets you pick per-model in `~/.pie/config.toml`:

| `type` | Code | Use it for |
|---|---|---|
| `native` | `pie/src/pie_driver/` | The default. Pie's own batched scheduler, FlashInfer / Metal kernels, full inferlet feature set (custom forward pass, page-level cache control, adapters, Zo). |
| `vllm` | `pie/src/pie_driver_vllm/` | Delegate inference to vLLM. Mature decode kernels, broad model coverage, vLLM's spec-decoding. |
| `sglang` | `pie/src/pie_driver_sgl/` | Delegate to SGLang. Strong custom-mask support, spec-decoding via SGLang. |
| `dummy` | `pie/src/pie_driver_dummy/` | No model loaded; returns random tokens. For protocol / inferlet tests without a GPU. |

The `type` is a discriminator. Driver-agnostic fields
(device, tensor parallel, dtype) live in `[model.X.driver]`; driver-specific
knobs in `[model.X.driver.<type>]`.

## Picking a driver

```toml
# ~/.pie/config.toml

[model.default]
hf_repo = "Qwen/Qwen3-0.6B"

[model.default.driver]
type = "native"                # native | vllm | sglang | dummy
device = ["cuda:0"]
tensor_parallel_size = 1
activation_dtype = "bfloat16"
```

Or via the CLI:

```bash
pie config set model.default.driver.type vllm
```

The relevant Python extra has to be installed for non-native drivers:

```bash
pip install "pie-server[cu128,vllm]"     # vLLM
pip install "pie-server[sglang]"         # SGLang
```

## Universal driver fields

Every driver consumes these from `[model.X.driver]`:

| Key | Default | Description |
|---|---|---|
| `type` | — | Which driver to load. Required. |
| `device` | — | List of devices, e.g. `["cuda:0"]` or `["cuda:0", "cuda:1"]`. Required. |
| `tensor_parallel_size` | `1` | Devices in one TP group. The remaining devices are used for DP. |
| `activation_dtype` | `"bfloat16"` | Activation precision. |
| `random_seed` | `42` | Deterministic seed for the worker. |

## Native driver

`type = "native"`. Pie's first-party inference path.

```toml
[model.default.driver.native]
gpu_mem_utilization = 0.8       # fraction of free GPU memory
max_batch_tokens   = 10240      # max tokens per fire_batch
max_batch_size     = 512        # max sequences per fire_batch
kv_page_size       = 16         # KV cache page size (tokens)
weight_dtype       = "auto"     # auto / float16 / bfloat16 / int4 / int8 / float8
use_cuda_graphs    = true       # compile + capture decode graphs
```

The native driver implements the full inferlet feature set:

- Custom forward pass with arbitrary attention masks ([custom forward pass](./writing-inferlets/custom-forward-pass)).
- Page-level cache control + page-trim ([KV cache](./writing-inferlets/kv-cache), [page-trim](./writing-inferlets/page-trim)).
- All sampler variants including raw-logits / logprobs / entropy ([sampler outputs](./writing-inferlets/sampler-outputs)).
- LoRA adapters and zeroth-order optimization ([adapters](./writing-inferlets/adapters)).

Quantization options are wired through `weight_dtype`. The float types
load full-precision weights; `int4`/`int8`/`float8` go through the
quantization path in `pie_driver/quantization.py`.

## vLLM driver

`type = "vllm"`. Pie wraps vLLM's `EngineArgs`; field names mirror vLLM
verbatim:

```toml
[model.default.driver.vllm]
attention_backend       = "FLASHINFER"   # FLASH_ATTN / TRITON_ATTN / FLEX_ATTENTION / …
gpu_memory_utilization  = 0.85
max_num_seqs            = 256
max_num_batched_tokens  = 8192
block_size              = 16             # KV block size; vllm picks if unset
enforce_eager           = false          # disable CUDA graphs

# n-gram speculative decoding (driver-supplied drafts)
spec_ngram_enabled      = false
spec_ngram_num_drafts   = 4
spec_ngram_min_n        = 2
spec_ngram_max_n        = 4
```

Trade-offs: vLLM has the broadest model zoo and best decode-batch
throughput on stock setups. Some inferlet features that depend on raw
forward-pass control (custom attention masks, page-trim, raw-logits
samplers) are not available — they require the native driver.

## SGLang driver

`type = "sglang"`. Mirrors SGLang's `ServerArgs`:

```toml
[model.default.driver.sglang]
attention_backend       = "triton"       # triton / flashinfer / fa3 / fa4 / …
mem_fraction_static     = 0.65
page_size               = 16
disable_cuda_graph      = false
cuda_graph_max_bs       = 256            # cap captured graph batch size
max_running_requests    = 256
max_total_tokens        = 65536
chunked_prefill_size    = 8192
kv_cache_dtype          = "auto"
trust_remote_code       = true
disable_radix_cache     = true           # pie owns prefix sharing
context_length          = 8192

# Pinned-host KV pool for D2H/H2D swap (GiB). 0 = disabled.
cpu_mem_budget_in_gb    = 0

# Adapter / Zo (CMA-ES). Forces disable_cuda_graph = true.
enable_adapter          = false
```

Trade-offs: SGLang has clean custom-mask support (hence Pie's recent
spec-decoding work runs through it) and `triton` works on any NVIDIA
SM 7.5+. Like vLLM, some native-only features are off the menu.

## Dummy driver

`type = "dummy"`. No weights loaded, no real compute — returns random
token IDs. Use it to exercise the engine + protocol path without a GPU,
or to bench the runtime independently of model load:

```toml
[model.default.driver]
type = "dummy"
device = ["cpu"]

[model.default.driver.dummy]
max_batch_tokens = 4096
max_batch_size   = 128
kv_page_size     = 16
```

You won't get useful generations — only the runtime is exercised.

## Choosing

- **Default → native.** Full feature set, best fit for advanced inferlets.
- **Need a model native doesn't support yet → vLLM**, if it's in the vLLM zoo.
- **Need spec-decoding or strong custom-mask support → SGLang.**
- **CI / runtime tests / no GPU → dummy.**

The driver registry lives in `pie/src/pie/drivers.py`. Each driver's
`__init__.py` calls `register_driver(DriverSpec(...))` at import time, so
adding a new driver is mostly: write the engine, register the spec.
